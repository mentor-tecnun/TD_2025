# D3.js
## Visualización de datos con SVG en el navegador

Librería JavaScript para unir **datos** con el **DOM**,
renderizando gráficos SVG completamente a medida.

---

## ¿Qué es D3.js?

- **D**ata-**D**riven **D**ocuments: el DOM cambia según los datos
- Trabaja directamente con SVG (y HTML, Canvas)
- Sin abstracciones: tú controlas cada pixel
- ~87 KB minificado, modular desde v7

> D3 no es una librería de "gráficos": es una herramienta para
> **unir datos con elementos del DOM** de forma declarativa.

---

## Instalación via CDN

```html
<script src="https://cdn.jsdelivr.net/npm/d3@7"></script>
```

El objeto global `d3` expone todos los módulos.

---

## Concepto clave: `select` y `selectAll`

D3 envuelve elementos del DOM igual que jQuery,
pero añade el concepto de **selección encadenada**:

```js
// Seleccionar un elemento
d3.select('#miSVG')           // como document.querySelector

// Seleccionar varios
d3.selectAll('circle')        // como document.querySelectorAll

// Encadenar atributos y estilos
d3.select('circle')
  .attr('cx', 100)
  .attr('cy', 50)
  .attr('r', 20)
  .style('fill', 'steelblue');
```

> Puedes hacer `d3.select(elementoDOM)` con cualquier nodo existente.

---

## Concepto clave: `.append()` y `.attr()`

D3 puede **crear** elementos SVG y configurarlos en cadena:

```js
const svg = d3.select('#contenedor')
  .append('svg')        // crea <svg> dentro de #contenedor
  .attr('width', 400)
  .attr('height', 200);

svg.append('rect')
  .attr('x', 10)
  .attr('y', 10)
  .attr('width', 80)
  .attr('height', 50)
  .attr('fill', '#4f46e5');

svg.append('text')
  .attr('x', 50)
  .attr('y', 80)
  .attr('text-anchor', 'middle')
  .text('Hola D3');
```

---

## Concepto clave: Data Join — `.data()` y `.join()`

D3: **vincula un array de datos a elementos DOM**.

```js
const datos = [10, 40, 25, 60, 30];

svg.selectAll('rect')         // selección vacía (no existen aún)
  .data(datos)                // vincular datos
  .join('rect')               // crear un <rect> por cada dato
    .attr('x', (d, i) => i * 50)   // d = dato, i = índice
    .attr('y', d => 100 - d)
    .attr('width', 40)
    .attr('height', d => d)
    .attr('fill', 'steelblue');
```

> Cada función `d => ...` recibe el dato vinculado al elemento.
> Esto es el patrón central de D3.

---

## Concepto clave: Escalas

Las escalas **mapean** un dominio de datos a un rango visual.
Son funciones: `escala(dato) → pixeles`

```js
// Escala lineal: números a píxeles
const y = d3.scaleLinear()
  .domain([0, 100])   // rango de datos
  .range([200, 0]);   // rango de píxeles (invertido: SVG crece hacia abajo)

// Escala de banda: categorías a posición x
const x = d3.scaleBand()
  .domain(['Ene', 'Feb', 'Mar'])
  .range([0, 300])
  .padding(0.2);      // espacio entre barras

// Uso
y(50)    // → 100  (mitad del alto)
x('Feb') // → posición x de la barra 'Feb'
x.bandwidth() // → ancho de cada barra
```

---

## Concepto clave: Ejes

D3 genera ejes SVG a partir de una escala:

```js
const escalaY = d3.scaleLinear().domain([0, 100]).range([200, 0]);
const escalaX = d3.scaleBand().domain(['Ene','Feb','Mar']).range([0,300]);

// Crear grupo <g> y llamar al eje
svg.append('g')
  .attr('transform', 'translate(40, 0)')   // margen izquierdo
  .call(d3.axisLeft(escalaY));             // eje vertical

svg.append('g')
  .attr('transform', 'translate(40, 200)') // moverlo al fondo
  .call(d3.axisBottom(escalaX));           // eje horizontal
```

> `.call(fn)` invoca `fn(seleccion)` — patrón idiomático de D3.

## Estructura básica de un gráfico D3

```html
<!DOCTYPE html>
<html>
<head>
  <script src="https://cdn.jsdelivr.net/npm/d3@7"></script>
</head>
<body>
  <div id="grafico"></div>
  <script>
    const margin = { top: 20, right: 20, bottom: 30, left: 40 };
    const width  = 400 - margin.left - margin.right;
    const height = 200 - margin.top  - margin.bottom;

    const svg = d3.select('#grafico').append('svg')
      .attr('width',  width  + margin.left + margin.right)
      .attr('height', height + margin.top  + margin.bottom)
      .append('g')
      .attr('transform', `translate(${margin.left},${margin.top})`);

    // Aquí van escalas, ejes y elementos visuales
  </script>
</body>
</html>
```

## Animaciones con `.transition()`

```js
// Transición básica
svg.selectAll('rect')
  .transition()
  .duration(800)           // ms
  .ease(d3.easeCubicOut)   // función de easing
  .attr('height', d => H - y(d.valor))
  .attr('y', d => y(d.valor));

// Retrasos escalonados (stagger)
svg.selectAll('rect')
  .transition()
  .delay((d, i) => i * 100)   // cada barra con 100ms más
  .duration(600)
  .attr('opacity', 1);
```

> Las transiciones interpolan **attrs, styles y transforms** automáticamente.
> No necesitas `requestAnimationFrame` ni nada externo.

## Eventos e interactividad

```js
svg.selectAll('rect')
  .on('mouseover', function(event, d) {
    // `this` = el elemento SVG, `d` = el dato vinculado
    d3.select(this)
      .transition().duration(150)
      .attr('fill', '#f43f5e');

    tooltip.style('display', 'block')
      .style('left', event.pageX + 10 + 'px')
      .style('top',  event.pageY - 20 + 'px')
      .text(`Valor: ${d.valor}`);
  })
  .on('mouseout', function() {
    d3.select(this).transition().duration(150).attr('fill', 'steelblue');
    tooltip.style('display', 'none');
  });
```

> `event` es el MouseEvent estándar del navegador.
> `d` es el dato que D3 vinculó a ese elemento con `.data()`.

## Actualizar datos (enter / update / exit)

El patrón completo de D3 para datos que cambian:

```js
function actualizar(nuevosDatos) {
  const barras = svg.selectAll('rect')
    .data(nuevosDatos, d => d.id);  // key function: identifica elementos

  // EXIT — eliminar elementos sobrantes
  barras.exit()
    .transition().duration(300)
    .attr('opacity', 0)
    .remove();

  // ENTER — añadir nuevos elementos
  // UPDATE — los ya existentes
  barras.join(
    enter  => enter.append('rect').attr('opacity', 0)
                   .call(e => e.transition().attr('opacity', 1)),
    update => update,
    exit   => exit.remove()
  ).attr('x', d => x(d.categoria))
   .attr('height', d => H - y(d.valor));
}
```

    exit=>exit.transition().duration(300).attr('opacity',0).remove()
  );
}

## Buenas prácticas

- Usa `margin convention` (top/right/bottom/left) siempre
- Define `width` y `height` del área interior (sin márgenes)
- Usa escalas en lugar de calcular posiciones a mano
- Guarda la referencia del `svg` interior (`<g>` con transform)
- Usa `key function` en `.data(arr, d => d.id)` al actualizar
- Prefiere `.join()` sobre el patrón manual enter/update/exit

## D3 vs Chart.js — ¿cuándo usar cada uno?

| | Chart.js | D3.js |
|---|---|---|
| Curva de aprendizaje | Baja | Media-alta |
| Gráficos estándar | Rápido | Más verboso |
| Uso de HTML | Canvas | SVG |
| Animaciones custom | Limitado | Total |
| Layouts | Standar | Standar y otros(treemap, force, geo...) |
| Ideal para | Dashboards rápidos | Visualizaciones a medida |
